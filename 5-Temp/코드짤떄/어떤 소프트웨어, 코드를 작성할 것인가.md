## 지속가능한 코드
- 이해하기 쉬운 코드
	- 적절한 길이
		- 한 파일에 300라인이 넘어가지 않는다
	- OOP
		- 적절한 책임이 있어야한다
- 모듈화
	- 버전업시 비즈니스 로직에 변경이 적다
- 테스트
	- 수정시 기능을 만족하고 있다는것을 알 수 있다


## 좋은 서비스
- 좋은 기능
	- 우선 좋은 기능이 있어야 사용자들이 사용한다
- 적은 에러
	- 기능을 사용하려는데 에러가 있으면 사용하지 않는다
- 적절한 응답시간
	- 응답시간이 짧아야 경험이 좋아진다
